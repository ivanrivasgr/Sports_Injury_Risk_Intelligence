I built a production-grade data pipeline for sports injury risk analysis.

Not to prove anything about pitches or players.
To show what it takes to ask the question rigorously.

Here's what I designed end-to-end 👇

𝗧𝗵𝗲 𝗽𝗿𝗼𝗯𝗹𝗲𝗺 𝗳𝗿𝗮𝗺𝗶𝗻𝗴

Most sports injury analyses skip this step entirely.

Before writing a single line of code, I defined:
→ The structural question (≠ "does the pitch cause injuries?")
→ What is actually observable vs what's a proxy
→ Which assumptions are required to proceed
→ What model outputs actually represent

That framing governs every decision downstream.

──────────────────────────────────────

𝗧𝗵𝗲 𝗮𝗿𝗰𝗵𝗶𝘁𝗲𝗰𝘁𝘂𝗿𝗲

A full Medallion stack:

🥉 Bronze — immutable raw storage, schema-on-read, timestamped ingestion
🥈 Silver — entity resolution, schema enforcement, DQ checks
📦 Enriched — cross-source joins, temporal alignment
🥇 Gold — rolling workload features, congestion index, injury labels
🧠 Feature Store — point-in-time correct, versioned, train/infer consistent

Each layer has data contracts. No layer overwrites the previous.

──────────────────────────────────────

𝗧𝗵𝗲 𝗱𝗮𝘁𝗮

Real Madrid injury records from Transfermarkt (2021–2025).
88 documented events. 25 players. 4 seasons.

69% muscular injuries.
Peak injury months: October–November and February–March.
Players with most injuries: Mendy (9), Camavinga (7), Courtois (6).

Every limitation is documented:
— ~30-40% of injuries are never publicly reported
— Venue is approximate (last match before report date)
— Training injuries are completely unobservable
— Association ≠ causation (stated explicitly, repeatedly)

𝗪𝗵𝗮𝘁 𝗜 𝗯𝘂𝗶𝗹𝘁

→ 10-page Streamlit app covering the full system
→ DAG with idempotent, partitioned, retry-safe tasks
→ Observability layer: DQ checks, PSI drift detection, alert rules
→ ML design: binary classifier + survival model options (no causal claims)
→ Live scraper for Transfermarkt + Open-Meteo (runs locally with network)
→ Full data lineage from raw ingest to feature store

The whole thing runs in Streamlit Cloud. Code is public on GitHub.
